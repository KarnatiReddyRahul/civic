# Contributing Guidelines

Thank you for your interest in contributing to CivicAssist AI.

## How to Contribute

### 1. Fork Repository

Create a personal fork of the project.

### 2. Create Branch

```bash
git checkout -b feature/feature-name
```

### 3. Make Changes

Implement your feature or bug fix.

### 4. Commit Changes

```bash
git commit -m "Added feature description"
```

### 5. Push Changes

```bash
git push origin feature/feature-name
```

### 6. Create Pull Request

Submit a pull request for review.

---

## Coding Standards

### Python

* Follow PEP8 guidelines.
* Use meaningful variable names.
* Add comments where necessary.

### API Development

* Maintain RESTful standards.
* Validate inputs.
* Handle exceptions gracefully.

### Frontend

* Keep UI clean and responsive.
* Follow accessibility best practices.

---

## Bug Reports

Please include:

* Bug description
* Steps to reproduce
* Expected behavior
* Screenshots (if applicable)

---

## Feature Requests

When requesting a feature:

* Describe the problem
* Explain the proposed solution
* Mention expected benefits

---

## Code Review Process

All contributions are reviewed before merging.

Review Criteria:

* Code quality
* Documentation
* Functionality
* Security

Thank you for contributing to CivicAssist AI.
